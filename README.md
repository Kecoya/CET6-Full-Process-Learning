## 📖 项目简介

本项目是一套**纯前端、零依赖、离线可用**的 CET-6（大学英语六级考试）综合备考工具。所有应用均为独立 HTML 文件，双击即可在浏览器中打开使用，无需安装任何软件或依赖。

### 核心特性

| 特性 | 说明 |
|:---|:---|
| 🎯 **全流程覆盖** | 从词汇筛选 → 语境学习 → 听力训练 → 翻译准备，完整覆盖备考全链路 |
| 🔊 **智能语音合成** | 基于 Web Speech API，支持男女声分离、语速调节、自动重试 |
| 📝 **真实模拟** | 听力应用完整还原 CET-6 考试播放流程和答题体验 |
| 📊 **数据驱动** | 所有学习数据均为本地 JSON/TXT 文件，完全掌控自己的学习进度 |
| 🖥️ **零配置** | 无需安装、无需构建、无需服务器，双击 HTML 即可开始学习 |
| 📱 **响应式** | 适配桌面、平板、手机多种设备 |

### 5 🎯 听力专项训练

**目录**：`听力专项训练/`（Flask 后端 + 拆分前端，是项目首个接入在线 AI 的模块）

以"**理解力**"为核心的听力专项训练：由 AI 现场生成与真题高度一致的听力原文，通过"盲听 → 自评理解 → 盲选 → AI 评分 → 精听"的闭环，训练并量化你的真实听力理解程度。

> 该模块采用 **Python(Flask) 后端 + HTML/JS/CSS 前端** 架构：命题/评分提示词、DeepSeek 调用、JSON 校验、落盘保存都在后端完成，**API Key 只存在服务端 `.env`，绝不进入浏览器**；前端专注界面与 Web Speech 播放。

#### 目录结构

```
听力专项训练/
├── app.py              # Flask：提供页面 + /api/generate、/api/evaluate、/api/save、/api/health、/api/types
├── requirements.txt    # Flask, requests
├── .env.example        # 复制为 .env 并填 DEEPSEEK_API_KEY
├── prompts/            # 📝 软编码提示词（随时改、随时生效，无需重启）
│   ├── system_generate.txt   # 生成原文+题目的系统提示词（命题总规则/选项设计/输出格式）
│   ├── system_evaluate.txt   # AI 评分的系统提示词
│   ├── types.json            # 三题型的显示信息（名称/emoji/题数/词数）
│   ├── spec_conversation.txt # 长对话命题细节（考点/干扰项/典型题干）
│   ├── spec_passage.txt      # 听力篇章命题细节
│   └── spec_lecture.txt      # 讲话·报道·讲座命题细节
├── templates/
│   └── index.html      # 页面结构
└── static/
    ├── app.js          # 状态机 / 渲染 / TTS 播放 / 逐句切句（可独立 node --check）
    └── style.css       # 样式
```

#### 自定义命题（软编码提示词）

所有命题/评分提示词都在 `听力专项训练/prompts/` 目录的**纯文本文件**里，**不写死在代码中**。后端每次生成/评分都会重新读取这些文件，所以：

- 用任意编辑器修改 `spec_*.txt`（题型命题细节）、`system_generate.txt`（总规则）或 `types.json`（题型信息），**保存后立即生效**——下一次生成就用新内容，无需改代码、无需重启服务器。
- 若删除某个文件，后端会自动回退到 `app.py` 里的内置默认值，不影响运行。
- 注意：改 `system_generate.txt` 里的 JSON 字段结构时，字段名要和 `app.py: validate_exercise` 一致（`dialogue` / `questions` / `options A-D` / `answer`），否则后端会判定生成失败并自动重试。

#### 快速开始

```bash
cd 听力专项训练
pip install -r requirements.txt        # Flask, requests（已装可跳过）
copy .env.example .env                 # Windows；macOS/Linux 用 cp
#   编辑 .env 填入 DEEPSEEK_API_KEY=sk-...
python app.py                          # 自动打开 http://127.0.0.1:5000
```

#### 训练流程

```
① 选择题型/AI生成或自定义原文
        ↓
② 第一遍盲听 → 写下理解 → 第二遍盲听 → 写下理解
        ↓
③ 盲选选项（题干隐藏，仅凭听力作答）
        ↓
④ AI 评分（DeepSeek 评价理解程度，0–100 分 + 点评）
        ↓
⑤ 逐句精听（开放原文，按句点击播放）
```

#### 功能亮点

| 功能 | 说明 |
|:---|:---|
| AI 生成原文 | 选择长对话/短文听力/讲座讲话三种题型，AI 生成与六级真题**高度一致**的原文（词数、词汇难度、句式复杂度均贴近真题）+ 题目与解析 |
| 自定义原文 | 可粘贴自己的英文原文，AI 据此生成选项、正确答案与解析（原文保留不变） |
| 两遍盲听 | 隐藏原文与背景信息，专注 listening；每遍结束后在对话框写下自己的理解 |
| 盲选选项 | 两遍盲听后展示各题 A/B/C/D 选项，**题干隐藏**，仅凭听力单选正确答案 |
| AI 理解度评分 | 点击评分，后端把原文 + 两遍理解文本 + 作答发给 DeepSeek，返回 0–100 分、主旨/细节/词汇点评、遗漏点与可执行提升建议 |
| 逐句精听 | 开放原文并按句拆分，可随时点击任意句子**单句播放**或"从此处连续播放" |
| 一键保存 | 评分后可把本次练习导出为 `my/YYMMDD听力专项.txt`（与项目其它听力数据同格式） |
| 三种题型规格 | 长对话（约 300–345 词 / 4 题 / W/M 交替）、听力篇章（约 240–265 词 / 3 题）、讲话·报道·讲座（约 345–430 词 / 3 题，CET-6 最高权重）。**词数与考点均由 2016–2023 年真题语料统计校准** |

#### 使用前提

- 需配置 **DeepSeek API Key**：复制 `.env.example` 为 `.env` 并填入 `DEEPSEEK_API_KEY`（密钥仅存于本机服务端，不会发往浏览器）。
- 需运行后端 `python app.py`（会自动打开浏览器）；语音播放仍使用浏览器 Web Speech API，建议 Chrome / Edge。
- 未配置 Key 时，页面顶部状态栏会提示"未配置"，生成/评分请求返回清晰错误而非崩溃。

### 6 📚 阅读写作翻译命题提示词

**目录**：`阅读写作翻译/prompts/`

一套**真题数据驱动**的命题系统提示词（system prompt），覆盖六级阅读、写作、翻译共**五大题型**。每条提示词的规格（词数、题数、题干句式、干扰项类型）都由 `材料/CET6真题提取数据/` 下 49 套真题（2019–2024）逐项统计得出，确保 AI 生成的题目与真题"相差无几"。

> 这些提示词目前为**独立可复用资产**：每个 `.txt` 即一个自包含的 system prompt，可直接喂给 DeepSeek / 任何 OpenAI 兼容模型（`response_format: json_object`）生成 JSON 题目。后续可仿照听力专项训练的 `load_*` 软编码模式接入后端、做成带训练流程的 app。

#### 提示词清单

| 文件 | 题型 | 真题实测规格（关键） |
|:---|:---|:---|
| `system_cloze.txt` | 选词填空 Section A | 原文 ~282 词 / 10 空 / 词库固定 15 词(A–O)，**必含同根不同形陷阱**（concede/conceded、correlation/correlate… 真题 71 组同根词族） |
| `system_matching.txt` | 长篇阅读 Section B（匹配） | 原文 **~1233 词** / 10–14 段 / 10 条陈述句（每条 ~17 词）对段落同义改写 |
| `system_careful.txt` | 仔细阅读 Section C | 每篇 **~447 词** / 5 题 / 题干 **What 占 ~80%** / 选项中位 8 词 |
| `system_writing.txt` | 写作 | **全部 150–200 词议论文**；4 类主题模板（importance of… / saying 谚语 / why students should… / 时代话题） |
| `system_translation.txt` | 翻译（汉译英） | **~174 中文字**；题材集中中国文化/国情/工程成就；专有名词首次出现括注英文 |

每个提示词的输出 JSON 都含**答案 + 逐题解析**（选词填空的词性线索、长篇匹配的定位词+改写、仔细阅读的定位句+干扰项错因、写作的提纲+高级表达、翻译的术语译法+难句拆解）。详见 `阅读写作翻译/prompts/README.md`。

## 🚀 快速开始

### 环境要求

| 要求 | 说明 |
|:---|:---|
| 浏览器 | Chrome、Edge、Safari 等支持 Web Speech API 的现代浏览器 |

### 使用步骤

#### 路径 A：听力练习

1. 双击打开 `CET-6 听力练习应用.html`
2. 点击 **"📂 加载练习数据"**，选择一个听力练习 JSON 文件（参见 [数据格式说明](#听力练习数据-json)）
3. （可选）点击 **"📂 加载生词列表"**，选择一个生词 TXT 文件
4. 在下拉框中选择女声和男声，点击 **🔊 试听** 确认声音
5. 点击 **"▶️ 完整播放"** 开始模拟考试，或逐行点击 🔊 单独播放
6. 作答后点击 **"提交答案"** 查看得分和解析

#### 路径 B：词汇筛选

1. 双击打开 `单词筛选应用.html`
2. 点击 **"📁 选择词汇文件"**，选择一个词汇 TXT 文件（如 `CET_4+6_edited.txt`）
3. （可选）点击 **"✓ 选择已认识单词文件"**，导入之前的 `known_words.txt`
4. 在 10 个单词卡片中，点击自己**不认识**的单词（变为红色）
5. 点击 **"✗ 标记为不懂"** 提交本批分类
6. 重复上述步骤直到所有单词处理完毕，或点击 **"💾 立即导出文件"** 保存进度

#### 路径 C：句子播放

1. 双击打开 `播放句子.html`
2. 点击 **"📂 加载句子数据"**，选择一个句子练习 JSON 文件（参见 [数据格式说明](#句子播放数据-json)）
3. 分别选择女声、男声和中文声音，点击试听确认
4. 调整语速和句间停顿时间
5. 点击 **"▶️ 全部播放"** 开始四遍循环学习，或逐句点击 🔊 单独播放

#### 路径 D：翻译词汇

1. 双击打开 `翻译单词学习.html`
2. 浏览各分类的高级表达替换
3. 点击任意表格行可复制高级表达到剪贴板
4. 点击右下角 **"🖨️ 打印"** 按钮生成打印版

## 📋 数据格式说明

### 听力练习数据 (JSON)

```json
{
  "title": "2024年6月六级听力真题 Section A",
  "topic": "日常对话",
  "context": "Two students are discussing...",
  "dialogue": [
    {"speaker": "W", "content": "Have you heard about the new policy?"},
    {"speaker": "M", "content": "Yes, I think it's going to affect everyone."},
    {"speaker": "W", "content": "That's exactly what I was thinking."}
  ],
  "questions": [
    {
      "question": "What are the speakers mainly discussing?",
      "options": {
        "A": "A new campus policy",
        "B": "A course schedule change",
        "C": "A job opportunity",
        "D": "A travel plan"
      },
      "answer": "A",
      "explanation": "对话中两人围绕新政策展开讨论……"
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `title` | string | 练习标题 |
| `topic` | string | 话题分类 |
| `context` | string | 背景信息 |
| `dialogue` | array | 对话数组，`speaker` 为 `"W"`（女）或 `"M"`（男） |
| `questions` | array | 题目数组，含 `question`/`options`/`answer`/`explanation` |

### 句子播放数据 (JSON)

```json
[
  {
    "covered_words": ["abstract", "concept"],
    "sentence": "The abstract concept of justice is difficult to define.",
    "translation": "正义这个抽象概念很难定义。"
  },
  {
    "covered_words": ["abandon"],
    "sentence": "She abandoned her plan to study abroad.",
    "translation": "她放弃了出国留学的计划。"
  }
]
```

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `covered_words` | string[] | 本句包含的生词列表 |
| `sentence` | string | 英文句子 |
| `translation` | string | 中文翻译 |

### 词汇文件 (TXT)

纯文本格式，每行一个单词：

```
abandon
ability
abstract
academic
```

### 生词文件 (TXT)

同词汇文件格式，每行一个单词，由单词筛选应用导出或手动创建。

## ❓ 常见问题

### 通用问题

**Q: 需要安装什么软件吗？**
A: 不需要。所有应用都是独立的 HTML 文件，使用支持 Web Speech API 的现代浏览器（推荐 Chrome 或 Edge）直接打开即可。

**Q: 可以离线使用吗？**
A: 首次使用需要联网以加载浏览器语音数据。语音数据下载完成后，即可离线使用所有功能。

**Q: 支持哪些浏览器？**
A: 推荐使用 Chrome、Edge 或 Safari。需确保浏览器支持 Web Speech API。

### 听力练习

**Q: 进度条显示的时间准确吗？**
A: 进度条基于词数（约 2.5 词/秒，根据语速调整）加上所有间隔时间动态估算，实际播放时间可能有轻微偏差。

**Q: 如何添加更多听力练习？**
A: 按照 [听力练习数据格式](#听力练习数据-json) 创建 JSON 文件，然后通过"加载练习数据"按钮导入即可。

**Q: 生词高亮的原理是什么？**
A: 导入生词列表后，应用会扫描对话和题目文本，将匹配的单词以黄色背景高亮显示，帮助你在听力中关注这些生词。

### 词汇筛选

**Q: 首次点击发音没有声音？**
A: 这是正常现象，语音数据需要 1-2 秒加载时间。应用会自动重试（最多 2 次），后续点击会立即播放。

**Q: 如何添加更多语音选项？**
A: 浏览器自动加载系统中已安装的语音。安装更多语音包或更换浏览器可获得更多选项。

**Q: 中途关闭页面会丢失数据吗？**
A: 会的。建议定期点击"💾 立即导出文件"保存进度。导出的 `remaining_words.txt` 可用于下次断点续学。

### 句子播放

**Q: 四遍循环是什么？**
A: 每个句子自动按 **英文 × 2 → 中文 × 1 → 英文 × 1** 的顺序播放。英文部分男女声交替，第三遍自动切换为中文语音朗读翻译。这种模式有助于在语境中强化记忆。

**Q: 句子数据怎么制作？**
A: 按照 [句子播放数据格式](#句子播放数据-json) 创建 JSON 数组文件。`covered_words` 字段会在界面上标注出该句包含的目标生词。

## 🛠 技术实现

### 技术栈

| 技术 | 用途 |
|:---|:---|
| HTML5 + CSS3 | 响应式界面、动画、打印样式 |
| Vanilla JavaScript | 原生实现，无任何框架依赖 |
| Web Speech API | 浏览器原生语音合成（TTS） |
| File API | 本地文件加载与导出 |

### 核心设计模式

#### 类式架构

每个应用使用一个核心类封装所有逻辑：

```
CET-6 听力练习应用  →  事件驱动 + 状态机
单词筛选应用       →  VocabularyClassifier 类
句子播放应用       →  SentencePlayer 类
```

#### 状态管理

```javascript
// 单词筛选应用的词汇状态管理
allWords            = [];   // 全部词汇
remainingWords      = [];   // 未显示的词汇池
displayedWords      = [];   // 已显示过的词汇
knownWords          = [];   // 本次新认识的单词
unknownWords        = [];   // 标记为不懂的单词
existingKnownWords  = [];   // 原有已认识的单词
```

#### 语音合成优化

- **延迟修复**：首次发音前自动等待语音引擎就绪
- **错误重试**：播放失败自动重试，最多 2 次
- **状态反馈**：播放时显示 ⏳ 等待和 🔊 播放中动画
- **动态加载**：自动检测可用语音，智能默认选择美式英语女声

#### 洗牌算法

使用 Fisher-Yates 算法随机打乱词汇顺序，保证每个单词的学习位置随机且不重复。
